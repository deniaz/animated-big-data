Animated Force-Directed Layout

Time-based animation of a Hypergraph in D3's Force-Directed Layout implementation.

In this experiment the data structure to contain the chronological itemset data was designed and tested. The above display graph is drawn by D3 using its Force Layout.

Every 2.5 seconds the graph takes a step forward. If a visible itemset should be hidden, or an invisible itemset become visible, this is done by using transitions-based animation.

The animation of the hypergraph and its itemsets works quite well. Nevertheless, the results are far from perfect. For one, the hypergraph acts very twitchy. There are also too many overlapping links and nodes. As part of this experiment a collision detection and prevention was implemented. The results are not in the above drawn visualization as results got even worse. Reason for the behaviour can be found in the documentation of the student research project.
